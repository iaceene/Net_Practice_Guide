OSI Model

📋 OSI Model 7 Layers Diagram
Visual representation of the 7-layer OSI model

The OSI (Open Systems Interconnection) model is a conceptual framework that standardizes the functions of a telecommunication or networking system, dividing it into seven layers. This model helps in understanding how different network components interact to enable communication between devices, regardless of their underlying technology.

Key Concept: Created by ISO (International Organization for Standardization), think about layers like steps to communicate between devices. These steps or layers are 7, and when communication happens between two devices, the sender and receiver process these steps in reverse - if the sender goes from layer 7 to 1, the receiver goes from 1 to 7.

Data Transformation Process

🔄 Data Transformation Diagram
How data transforms at each layer

At Sender:

  • At each layer, a header can be added to the data unit
  • At layer 2, a trailer is added as well
  • When formatted data unit passes through physical layer, it is changed into an electromagnetic signal and transported along physical link

Encapsulation:

A packet (header and data) at level 5 is encapsulated in a packet in level 4, and so on. The data portion of packet at level N-1 carries the whole packet (data and header/trailer) from level N.

At Destination:

  • Data then moves back up through the layers
  • Headers and trailers attached to data at the corresponding layer are removed (decapsulated)
  • At layer 5, the message is again in form appropriate to the application and is made available to user

The 7 OSI Layers

1. Physical Layer

The physical layer is responsible for movements of individual bits from one hop (node) to the next.

Key Functions:
  • Physical characteristics: Defines the characteristics of the interface between devices and media
  • Representation of bits: Defines how 0s and 1s are changed to signals
  • Data rate: Defines the number of bits sent per second
  • Synchronization: Sender and receiver must use the same bit rate
  • Line configuration: Point-to-point or multipoint connections
  • Physical topology: Star, mesh, bus, ring, or hybrid
  • Transmission mode: Simplex, half-duplex, or full-duplex

2. Data Link Layer (Hop to Hop Delivery)

The data link layer is responsible for moving frames from one hop (node) to the next.

Key Functions:
  • Framing: Divide the stream of bits into data units called frames
  • Physical addressing: Adds header with sender and receiver addresses
  • Flow control: Prevents overwhelming the receiver
  • Error control: Detects and retransmits damaged or lost frames
  • Access control: Determines which device has control over the link
MAC Address Example: 07:01:02:01:2C:4B Ethernet uses 6-bytes (48-bits) physical address imprinted on the NIC

3. Network Layer (End to End Delivery)

The network layer is responsible for the delivery of individual packets from the source host to the destination host across multiple networks.

Key Functions:
  • Logical addressing: Adds unique IP addresses to packets
  • Routing: Provides routing mechanism for routers to route packets to final destination

Important: IP addresses are 32-bit addresses that uniquely define a host connected to the Internet. No two hosts on the internet can have the same IP address. The logical addresses remain the same throughout the journey, while physical addresses change from hop to hop.

5. Session Layer

The session layer is responsible for dialog control and synchronization.

Key Functions:
  • Dialog control: Allows communication in half or full duplex
  • Synchronization: Adds checkpoints for recovery in case of failure

6. Presentation Layer

The presentation layer is responsible for translation, compression, and encryption.

Key Functions:
  • Translation: Converts between sender and receiver formats
  • Encryption/Decryption: Ensures privacy and security
  • Compression: Reduces the number of bits for efficient transmission

7. Application Layer

The application layer is responsible for providing services to the user.

Services Include:
  • Telnet: Remote system login
  • HTTP: Web file transfers
  • SMTP: Electronic mail transfer
  • Mail services, File transfer and access, Remote log-in, Web access

TCP/IP Protocol Suite

The layers in the TCP/IP protocol suite do not exactly match those in the OSI model. However, when TCP/IP is compared to OSI, we can say that the TCP/IP protocol suite is made of five layers: physical, data link, network, transport, and application.

🌐 TCP/IP vs OSI Model Comparison
Visual comparison of TCP/IP and OSI models
TCP/IP Layers:
  • Physical and Data Link Layers: Handle hardware addressing and physical transmission
  • Network Layer: Handles logical addressing and routing (IP)
  • Transport Layer: Provides reliable data transfer (TCP/UDP)
  • Application Layer: Combines Session, Presentation, and Application from OSI

Most local-area networks use a 48-bit (6-byte) physical address written as 12 hexadecimal digits; every byte (2 hexadecimal digits) is separated by a colon:

07:01:02:01:2C:4B

A 6-byte (12 hexadecimal digits) physical address.

IP Addresses & Subnet Masks

Understanding Computer Networks

Computer networks can be defined as the exchange of network packets between computing machines across the world with the help of data lines like wire cables, optical fibers, etc.

🌍 Network Infrastructure Diagram
Global network connections and data flow

Common Network Terms

Nodes

Nodes in computer networks mean any computing device such as computers, mobile phones, tablets, etc which try to send and receive network packets across the network to another similar device.

Network Packets

Network packets are nothing but the information or units of data that a source node wants to send/receive to/from the destination node.

Internet Protocol (IPs)

An IP of a computer device is the address of that device in a computer network. Technically, it is a 32-bit number which identifies devices in a network.

IP Types:
  • IPv4: 32 bits (four bytes) - Example: 104.244.42.129
  • IPv6: Eight hexadecimal numbers separated by ":" - Example: 2001:0cb8:85a3:0000:0000:8a2e:0370:7334

IPv4 Address Classes

📊 IP Address Classes Chart
Visual breakdown of Class A, B, C, D, E addresses

Class A

  • First bit: Always "0"
  • Default subnet mask: 255.0.0.0
  • Network bits: 8, Host bits: 24
  • Range: 1.x.x.x to 126.x.x.x
  • Networks: 2^7 - 2 = 126
  • Hosts per network: 2^24 - 2 = 16,777,214

Class B

  • First two bits: "10"
  • Default subnet mask: 255.255.0.0
  • Range: 128.0.x.x to 191.255.x.x

Class C

  • First three bits: "110"
  • Default subnet mask: 255.255.255.0
  • Range: 192.0.0.x to 223.255.255.x

Static vs Dynamic IPs

  • Static: Remain constant for a device over time (servers, remote systems)
  • Dynamic: Change over time, assigned by DHCP servers

Network Infrastructure Components

Routers

Routers are devices that forward data packets between computer networks. They use routing tables to determine the best path for data transmission.

Routing Table Example: Destination Gateway Genmask Flags Metric Refs Iface default 192.168.0.1 0.0.0.0 UG 1024 233 eth0 192.168.0.0 * 255.255.255.0 UC 0 0 wlan0

Network Address Translation (NAT)

NAT is a technique used by routers to provide internet service to more devices with fewer public IPs. A router is assigned a single public IP by the ISP and assigns private IPs to connected devices.

NAT Process Example:

Home device (192.168.1.100:37641) → Router NAT → Public connection (104.244.42.129:59273) → Remote server (72.14.204.147:80)

DHCP (Dynamic Host Configuration Protocol)

DHCP is responsible for assigning dynamic IP addresses to hosts. The DHCP server is maintained by the ISP or router.

DNS (Domain Name System)

DNS servers maintain records of domain name to IP address mappings, allowing us to use memorable domain names instead of IP addresses.
